# ⚙️ InfraOps Lite Agent

> **One-Pager | Agente para Clientes | Prioridade: Wave 3**

---

## Problema do Cliente

Clientes precisam provisionar infraestrutura Azure de forma consistente, segura e
auditável. O processo atual é manual (portal) ou parcialmente automatizado com IaC
inconsistente. O projeto azure-agentic-infraops-accelerator (Jonathan Vella) demonstrou
o poder de multi-agent workflows para InfraOps, mas seu escopo (16 agentes + 11 subagentes)
é complexo demais para adoção rápida. CSAs precisam de uma versão simplificada que
demonstre o conceito e gere valor imediato.

---

## Solução

O **InfraOps Lite Agent** é uma versão simplificada do InfraOps Accelerator, focada em
3 passos: Requirements → Architecture Assessment → IaC Generation. Utiliza Azure Verified
Modules (Bicep/Terraform), aplica WAF assessment embedded, e mantém human-in-the-loop
approval gates. Ideal para demos, engagements rápidos e adoção gradual do conceito
agentic InfraOps.

---

## Como Funciona

```
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│ 1. REQUIRE    │────▶│ 2. ARCHITECT  │────▶│ 3. GENERATE   │
│ Requirements  │     │ Assessment &  │     │ IaC Code      │
│ Gathering     │  ✋ │ Design Review │  ✋ │ (Bicep/TF)    │
└───────────────┘     └───────────────┘     └───────────────┘
       ✋ = Human Approval Gate
```

| Fase | O que faz | Output |
|------|-----------|--------|
| **1. Requirements** | Coleta requisitos: workload, SLA, compliance, scale, networking | Requirements Doc |
| **2. Architecture** | Assessment WAF, design review, challenger questions | Architecture Decision + WAF Score |
| **3. IaC Generation** | Gera código IaC com Azure Verified Modules | Bicep/Terraform templates |

## Architecture Assessment (Fase 2)

### WAF Quick Assessment (embedded)
| Pilar WAF | Checks | Score |
|-----------|--------|-------|
| **Reliability** | AZ, DR, backup, SLA composition | /25 |
| **Security** | Network isolation, identity, encryption | /25 |
| **Cost** | Right-sizing, commitment opportunities | /25 |
| **Operational Excellence** | IaC, monitoring, alerts, tagging | /25 |
| **Performance** | Scaling strategy, caching, CDN | Bonus |

### Challenger Questions (Anti-pattern Detection)
| Anti-pattern | Challenger pergunta |
|-------------|---------------------|
| Over-engineering | "Você precisa de multi-region para esse SLA?" |
| Under-sizing | "500 req/s sustentado suporta os picos de Black Friday?" |
| Vendor lock-in | "Containers em AKS ou seria App Service suficiente?" |
| Cost blindness | "Estimou o custo de Cosmos DB com esse throughput?" |

## IaC Generation (Fase 3)

### Azure Verified Modules (AVM)
| Recurso | Módulo AVM | Configuração |
|---------|-----------|--------------|
| Resource Group | `avm/res/resources/resource-group` | Naming, tags, location |
| Virtual Network | `avm/res/network/virtual-network` | Address space, subnets, NSGs |
| AKS | `avm/res/container-service/managed-cluster` | Node pools, networking, monitoring |
| SQL Database | `avm/res/sql/server` | Geo-replication, backup, firewall |
| Key Vault | `avm/res/key-vault/vault` | Soft delete, purge protection, RBAC |
| App Gateway | `avm/res/network/application-gateway` | WAF, SSL, health probes |

### Exemplo de Output (Bicep)
```bicep
// Generated by InfraOps Lite Agent
// Requirements: Web app .NET 8 + SQL + Redis
// WAF Score: 78/100
// Human Approved: ✅

module vnet 'br/public:avm/res/network/virtual-network:0.5.0' = {
  name: 'vnet-app-prod'
  params: {
    name: 'vnet-app-prod-brazilsouth'
    addressPrefixes: ['10.0.0.0/16']
    subnets: [
      { name: 'snet-aks', addressPrefix: '10.0.1.0/24' }
      { name: 'snet-data', addressPrefix: '10.0.2.0/24' }
      { name: 'snet-appgw', addressPrefix: '10.0.3.0/24' }
    ]
  }
}
```

## Exemplo de Uso

```
CSA: @infraops-lite Preciso de IaC para um novo workload.
     Tipo: E-commerce web app
     Stack: .NET 8, AKS, Azure SQL, Redis Cache
     SLA: 99.95%
     Region: Brazil South
     IaC: Bicep
     Budget: R$ 45K/mês

Agent: ⚙️ InfraOps Lite — Requirements Phase

       📋 REQUIREMENTS CAPTURED:
       ├── Workload: E-commerce (.NET 8)
       ├── Components: AKS + Azure SQL + Redis + App Gateway
       ├── SLA: 99.95% → AZ-based deployment required
       ├── Security: Private endpoints, managed identity
       ├── Region: Brazil South
       └── Budget: R$ 45K/mês

       ✋ HUMAN APPROVAL NEEDED:
       Requirements look correct? [Yes/Refine]

CSA: Yes

Agent: ⚙️ InfraOps Lite — Architecture Assessment

       📐 ARCHITECTURE ASSESSMENT:
       ├── SLA Composition: 99.95% (achievable with AZ)
       ├── WAF Reliability: 21/25 ✅
       ├── WAF Security: 23/25 ✅
       ├── WAF Cost: 19/25 🟡
       │   └── Suggestion: Consider reserved capacity for SQL
       ├── WAF OpEx: 22/25 ✅
       └── Overall WAF Score: 85/100 🟢

       🤔 CHALLENGER QUESTIONS:
       ├── "AKS vs App Service: com 500 req/s, App Service
       │    seria suficiente e 30% mais barato?"
       └── "Redis Cache: Basic tier suficiente para session
            state, ou precisa de clustering?"

       ✋ HUMAN APPROVAL NEEDED:
       Proceed with AKS + Standard Redis? [Yes/Change]

CSA: Yes, keep AKS. Use Basic Redis.

Agent: ⚙️ InfraOps Lite — IaC Generation

       📦 IaC GENERATED (Bicep + AVM):
       ├── main.bicep (orchestration)
       ├── modules/
       │   ├── networking.bicep (VNet, subnets, NSGs)
       │   ├── aks.bicep (AKS cluster, node pools)
       │   ├── sql.bicep (Azure SQL, geo-rep, firewall)
       │   ├── redis.bicep (Redis Cache, Basic)
       │   ├── appgw.bicep (App Gateway + WAF)
       │   ├── monitoring.bicep (Log Analytics, alerts)
       │   └── keyvault.bicep (Key Vault, managed identity)
       ├── parameters/
       │   ├── prod.bicepparam
       │   └── dev.bicepparam
       └── README.md (deployment guide)

       💰 ESTIMATED COST: R$ 38K/mês (within R$ 45K budget)
       📂 PR created: repo/infraops-ecommerce#1
```
